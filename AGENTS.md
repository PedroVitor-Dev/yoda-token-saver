# Yoda Token Saver — Agent Instructions

This repository contains an open-source Codex Skill focused on token-efficient AI communication.

The project must remain:

- Professional.
- Useful.
- Minimal.
- Easy to understand.
- Open-source friendly.

The Yoda theme is part of the project identity, but clarity always comes first.

---

## Project Goal

Help AI assistants produce shorter responses while preserving:

- Meaning.
- Accuracy.
- Context.
- Practical value.

Main principle:

> Reduce tokens. Keep meaning.

---

## Development Guidelines

When working on this repository:

1. Keep files simple.
2. Avoid over-engineering.
3. Prefer plain Markdown.
4. Make examples easy to copy.
5. Keep documentation beginner-friendly.
6. Preserve the professional tone.
7. Use humor lightly and intentionally.

---

## Writing Style

Use:

- Short sentences.
- Clear headings.
- Practical examples.
- Direct explanations.
- Minimal jargon.

Avoid:

- Long paragraphs.
- Excessive jokes.
- Fake complexity.
- Buzzwords without purpose.
- Overly casual language in core documentation.

---

## Skill Behavior Rules

The skill should optimize AI responses by:

- Removing filler.
- Reducing repeated context.
- Shortening explanations.
- Preserving important details.
- Keeping answers actionable.

The skill must not remove:

- Safety warnings.
- Required technical steps.
- Important constraints.
- Error context.
- User-requested details.

---

## Repository Standards

Before changing the project, check:

- Does this make the skill more useful?
- Does this reduce unnecessary complexity?
- Does this keep the tone professional?
- Does this preserve the core idea?

If the answer is no, avoid the change.
